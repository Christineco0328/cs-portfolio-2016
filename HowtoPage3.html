How to Create Clickable Keyboard

Here I will show you how use Javascript, HTML, and CSS to create a Clickable Keyboard!

HTML for your Keyboard

Step 1: Create your Keys (Divs! Divs! Divs!)

In order to create a piano, we need to have keys to press. Therefore, in our HTML we can use divs to represent the keys on the keyboard. You will need to create a new div for each key you make, thus I suggest that you at least create 8 divs to complete a full octave on the piano.

Step 2: Give each Key an ID

In our javascript we will need to refer each key individually to assign specific sounds to each key according to their place on the keyboard. Thus, to make each key unique, we can give each div a different ID.

As mentioned in previous How to Pages , adding an ID will look like this:

<div id = "C" > </div>

Javascript for your Keyboard

Step 1: Get Element By Id

Through javascript, we will attach the different pitches found in the scale to each of the divs (keys). In order for us to interact with and manipulate the divs, we need to create a connection between the divs in our HTML and our javascript. We can achieve this using the method getElementByID().

This method takes the element in the HTML based on its ID and brings over its information onto the Javascript so that we can manipulate its data. Therefore, to bring over the data associated with the keys in our HTML, we will insert the ID of each of our divs into the getElementByID method.

Step 2: Add Event Listeners

Next, we want the program to be able to play a certain note when a key on our physical keyboard is pressed. In order for this to happen, we need our program has to know when the key is being pressed. Thus, we can use an Add Event Listener to enable our program to listen for when a key is pressed.

First, we need to tell the program which element it should be listening to. The program needs to be listening whenever the user is on the webpage. The element document refers to the entire HTML document. Thus, we can attach the add event listener to the page as a whole so that the add event listener is listening whenever the document is being used. Your code should now look like this:

document.addEventListener

Getting more specific, we now need to tell the program WHAT it needs to listen for. Using the event "keydown", the program will listen for when a key on the physical keyboard is pressed.

document.addEventListener("keydown", )

Once this add event listener hears the event, "keydown", it will record all the information about what happened when the key was pressed, including which keys were pressed. Any available information about the event will then be stored into an argument called "e".

Now that it knows what it has to listen for, the program now needs to know what it has to DO when it "hears" the keydown event. We will do this using a callback function--a function that is passed onto another function. This callback function, which we will simply call "function", will take in "e," which holds all the information about the event, as stated. This data, or "e" will later be refered to in our if statement. The format for your add event listener and callback function should now look like this.

document.addEventListener("keydown",function(e))

Step 3: If statements

For each key, we want a different sound to be played. Thus, for each div, we can use if statements to play a certain sound if a certain key is played--creating an association between the key on the scale to the div. To write our if statement, we will first use the keyword, if. Then, we need to create a condition statement that will check to see if the key pressed on the computer keyboard is a specific key. In order for the program to find this key, it has to look through e. However, e contains all the information associated with the event, but we only need to know which key is being pressed. Thus, to make the output more specific, we can use dot notation to tell the program to only look at the "keycode" of the key.

(Each key on the keyboard is associated with a certain keycode. Find out the keycode of any key Here )

Then, we also have to check to see if this information is equal to the keycode of the key that we want the sound to be associated. This can be done using three equal signs followed by the keycode of the desired key. Your if statement and condition should now look like this:

if (e.keyCode === 65)

Step 4: The Codeblock for your If Statement

Finally, we need to tell the program what to do if our condition is true. For the purposes of creating a keyboard, we want a certain note on the keyboard to play once a key is pressed. Because we have multiple events that trigger the same noise within this keyboard, we can put the audio files for each pitch on the scale into new variables within our global data. By establishing these variables in our global data, we give our entire code access to these files to be applied to multiple functions. This is the exact purpose of the play() method. The play method is used to start play either a video, or in our case, an audio file. We can then use dot notation to specify to our program to apply the "play()" method to the audio file contained in the variables we established earlier. In sum, if the if statement is true, the desired audiofile will be played using the "play()" method. Your if statement and code block combined should follow this format:

if (e.keyCode === *insert keycode of desired key on keyboard*)
